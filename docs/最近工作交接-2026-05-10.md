## 二、最近完成的两轮工作

### Round 1（commit `e40ac54`）：STDMA-inspired 基线

新增两个 paper-inspired、非学习 MAC 基线，作为 macMode 接入：

- `greedy_stdma`：每帧用 `occupancyTable` + 一/两跳邻居关系给时隙打 STDMA cost（一跳冲突 +100、两跳冲突 +10、远端节点 +1），按 cost 升序选 K 个时隙；启发式概率 Bernoulli 采样发起 RTS。
- `traffic_adaptive_tdma`：始终申请 owner slot；高压力（队列长度 ≥ `highLoadThreshold` 或 `lambdaEwma ≥ highLoadThreshold/2`）时按 STDMA cost 在剩余时隙追加 `ceil(Q/highLoadThreshold)` 个空间复用申请，剔除一跳冲突槽。

二者都注册为 `runner=omnet_only / needs_rl=false`，不启动 Python。详情：[docs/算法改进记录.md](算法改进记录.md) `## 新增 STDMA-inspired 基线（2026-05-10）`。

### Round 2（commit `44ac5e9`）：MANET 化重构（架构级改动）

把仿真器从"逻辑拓扑级抽象"升级为 **L1 等级 MANET 物理层抽象**：

| 维度 | 实现 | 文件 |

## 三、关键架构决策（动这些前先确认）

1. **`linkModel` 默认是 `distance`**（不是 `legacy_ned`）——意味着所有现有场景从这一版起按距离判定连通性。回归校验显示 N9_ring + commRange=150 + static mobility 复现历史 .ned ring 实验，PDR / goodput / MAC 效率误差 ≤ 2%。**如果用户要复现旧实验做对比，需要显式传 `--link_model legacy_ned`。**

2. **`mobilityMode` 默认是 `static`**（节点位置不变）——MANET 移动是 opt-in。需要时通过场景后缀启用：
   - `_manet` / `_manet_pedestrian`：行人速度 1–3 m/s, pause ≤5s
   - `_manet_vehicular`：车辆速度 5–15 m/s, pause ≤1s
   - `_manet_dense`：arena 300×300（密集）
   - `_manet_sparse`：arena 800×800（稀疏）

3. **`Network.ned` 是全互连**（每对 `i!=j` 都有 gate）——运行期 `isActiveLink` 过滤决定哪些链路真实可用。**不要把 Network.ned 改回硬编码连接**，否则 mobility 会因缺 gate 而失效。

4. **`topologyMode` 语义改了**：从"NED 连接形状"变为"初始几何布局"。`layoutInitialPositions()` 按 0.7×commRange 自适应 ring 半径 / grid 间距 / star 外环。这是与历史基线兼容的关键校准。

5. **互斥校验**：`mobilityMode=random_waypoint` + `dynamicTopologyMode=topology_switch` 不兼容（位置 warp 与拓扑切换打架），`initialize()` 中会强制把后者改回 `static` 并打 WARNING。

6. **`omnetpp.ini` 含用户预先存在的 tweaks**：在 Round 2 之前用户已修改了 3 处（`sim-time-limit=15000s`、`seed-set=2`、`adaptiveMultiplier=true`）。这些是用户的实验调整，不是 STDMA/MANET 工作的副作用，但 Round 2 commit 把它们一并入库了（避免脏工作树拖到下一轮）。

## 五、常见陷阱（别踩）

1. **直接跑 `./DynamicTDMA -f omnetpp.ini -u Cmdenv` 会卡住**：默认 `macMode=dynamic_tdma` 会等 Python 管道。冒烟测试一定要传 `**.nodes[*].macMode = "plain_tdma"` 或类似 non-PPO macMode，参考 CLAUDE.md / AGENTS.md 中的 sanity 命令。

2. **不要 `git add -A` 或 `git add .`**：仓库根目录有 `.ablation_ini_backup_*`、`tdma_run_*.ini`、`.docx` 周报等，会污染 commit。Claude 已建立的习惯是按文件名精确 staging。

3. **不要直接编辑 `TDMA_Messages_m.cc/.h`**：由 `opp_msgc` 从 `TDMA_Messages.msg` 自动生成。

4. **不要改 `Network.ned` 回硬编码**：mobility 依赖全互连 gate。如果要恢复旧拓扑行为，使用 `--link_model legacy_ned`，**不要**回改 NED。

5. **`applyMobilityForFrame` 由 node 0 在每帧第一个时隙触发一次**：不要在其他位置（如 `handleData` 或 `scheduleRequests`）重复调用，会把 dt 算两次。

6. **commRange 校准与 N、topologyMode 关联**：默认 `commRange=150m` 是为 N=9 ring/star/grid + arena 500×500 标定的。若做 N=20+ 或 arena 显著变化的实验，需要重算 commRange，否则初始拓扑可能退化为全连通或全不连通。

7. **omnetpp.ini 已有用户预设的 sim-time-limit=15000s**：跑短冒烟时一定通过 `--sim_time` 覆盖（`run_joint.sh` 会写临时 ini，不会污染原 ini）。
