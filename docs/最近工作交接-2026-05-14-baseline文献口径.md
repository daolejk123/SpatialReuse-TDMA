# 最近工作交接 - 2026-05-14 - baseline 文献口径

## 本轮做了什么

接续 v4 主表结果收尾后，针对“当前对比算法是否有论文说服力、是否需要补真实论文算法”做文档更新，没有跑新仿真。

更新重点在 [论文指标与性能对比基准.md](论文指标与性能对比基准.md)：

- 增加 baseline 说服力分级。
- 明确当前 `zmac_like`、`trama_like`、`bandit_index_proxy` 只能写成 paper-inspired/proxy，不能写成完整复现。
- 补入真实论文候选：STDMA column-generation / greedy approximation、Z-MAC、TRAMA、DLMA。
- 给出论文主表与附录 baseline 口径建议。

## 当前判断

当前对比矩阵可支撑“本地强基线 + paper-inspired baseline + RL 消融”的论文雏形，但还不适合宣称“完整复现并超过经典协议”。

建议论文表述：

- 主表保留：`plain_tdma`、`heuristic_only`、`greedy_stdma`、`traffic_adaptive_tdma`、`ppo_baseline`、`B`。
- 附录/扩展表保留：`zmac_like`、`trama_like`、`bandit_index_proxy`、v4 starvation variant。
- 所有 `*_like` 方法统一写成 inspired/proxy。
- 若继续增强说服力，优先补 `offline_upper_bound` 或更规范的 `greedy_stdma_2hop`，而不是先做 B-MAC/S-MAC/T-MAC。

## 已确认真实论文方向

| 方向 | 代表论文 | 当前用途 |
|---|---|---|
| spatial-reuse / STDMA | Värbrand, Yuan, Björklund, INFOCOM 2003 | 支撑 `greedy_stdma` 与可选 upper-bound |
| Z-MAC | Rhee et al., IEEE/ACM ToN 2008 | 支撑 `zmac_like`，但只能写 inspired |
| TRAMA | Rajendran, Obraczka, Garcia-Luna-Aceves, SenSys 2003 / Wireless Networks 2006 | 支撑 `trama_like`，但只能写 inspired |
| DLMA | Yu, Wang, Liew, ICC 2018 | 适合作为 RL-MAC related work；实现成本较高 |

## 下一步建议

1. 若目标是尽快投稿，先保持当前实验矩阵，论文中严格使用 inspired/proxy 口径。
2. 若还有一轮开发时间，优先实现小规模 `offline_upper_bound` 或规范化 `greedy_stdma_2hop`，它最能增强 TDMA/空间复用方向的说服力。
3. 若审稿方向偏 RL-MAC，再考虑实现 DLMA-style DQN access baseline。
4. 当前没有真实能耗模型，不建议把 S-MAC/T-MAC/B-MAC 放入主表实验。
