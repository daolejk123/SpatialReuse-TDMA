# 实验运行登记

> 用途：记录正式实验、smoke、附录实验、敏感性实验的命令、结果目录、完整性检查和论文可用性。
> 规则：大型 `logs/`、`checkpoints/`、`out/` 不直接提交 Git；论文结论优先使用 `seeds=1 2 3` 的正式批次，单 seed 只作为 smoke 或初步对比。

## 状态枚举

- `planned`：已计划，尚未运行。
- `running`：正在运行。
- `completed`：运行完成，结果可分析，但尚未认定进入论文。
- `invalid`：参数、代码版本、训练预算或完整性不满足要求，不进入论文。
- `accepted_for_paper`：已复核，可作为论文主表或附录候选数据。

## 标准完整性检查命令

把 `<RESULT_DIR>` 替换为实际结果目录：

```bash
find <RESULT_DIR> -name run_status.tsv | wc -l
find <RESULT_DIR> -name python.log | wc -l
find <RESULT_DIR> -name python.log -exec grep -H "target_updates reached: 400" {} \; | wc -l
find <RESULT_DIR> -path '*/python.log' | grep -Ev '/(ppo_baseline|B)/' || true
git diff -- omnetpp.ini
```

正式 MANET 6 方法 × 2 场景 × 3 seeds 的预期：

```text
run_status.tsv = 36
python.log = 12
target_updates reached: 400 = 12
非 PPO python.log = 空
omnetpp.ini diff = 空
```
