## 当前论文主线

- 研究对象：面向 MANET/UAV-inspired 场景的动态 TDMA/STDMA 时隙调度。
- 主方法：OMNeT++ 在线仿真 + Python PPO/LSTM 调度器，核心学习方法在实验表中记为 `B`，同时保留 `ppo_baseline` 作为弱化 RL 对照。
- 当前主表候选：v3 正式 MANET 对比实验，目录 `logs/benchmark_manet_formal_compare_v3_20260510_151910`。
- v4 结论：饥饿惩罚能明显压低 sparse 场景 starvation，但 pedestrian 场景吞吐/PDR 回退，因此 v4 不直接替代 v3；更适合作为 starvation/fairness trade-off 或 reward 修复消融。
- 论文 baseline 口径：`*_inspired` 和 `*_like` 只能写作“论文启发式近似/本地复现”，不能宣称完整复现原论文协议。

## 关键实验结论

### v3 正式 MANET 主表候选

- 目录：`logs/benchmark_manet_formal_compare_v3_20260510_151910`
- 完整性：36/36 `run_status.tsv`；12/12 PPO/B 日志达到 `target_updates reached: 400`。
- 论文用途：当前主表候选，可作为论文主要性能表的基础。
- 口径：优先比较 `plain_tdma`、`heuristic_only`、`greedy_stdma`/`greedy_stdma_2hop`、`traffic_adaptive_tdma`、`ppo_baseline`、`B`。

### sparse200 附录与敏感性实验

- sparse200 附录目录：`logs/benchmark_manet_sparse200_appendix_server_b_20260510_203258`
- 论文用途：用于说明 sparse/UAV 场景下负载、队列与 starvation 的敏感性，不直接替代 v3 主表。

### 饥饿惩罚修复

- 关键参数：`starvationThresholdFrames=5`，`starvationPenaltyCoef=0.1`，`maxStarvationFramesForPenalty=20`。
- sparse200 修复目录：`logs/benchmark_manet_sparse200_starvation_fix_20260511_014614`
- 结论：在 sparse200 场景中，`B` starvation 约下降 70.7%，`ppo_baseline` starvation 约下降 75.0%；PDR 分别约提升 1.4% 与 22.9%；Wt_P95 均约下降 9.8%。

### v4 主表复核

- 目录：`logs/benchmark_manet_formal_compare_v4_20260511_104010`
- 完整性：36/36 `run_status.tsv`；12/12 PPO/B 达 400 updates；非 PPO 方法无额外 `python.log`；未发现 run_status 失败项。
- 关键收益：`N12_grid_manet_uav_sparse / B` starvation `1.2222 -> 0.3333`，Wt_P95 约下降 9%。
- 关键代价：`N12_grid_manet_pedestrian / B` goodput/PDR 约下降 20%；`ppo_baseline` 在两个主场景 PDR 均下降，但等待时延改善。
- 论文口径：v3 仍是主表候选；v4 作为 reward 修复、starvation/fairness trade-off 消融。

## 真实论文支撑方向

- STDMA/spatial-reuse TDMA：用于支撑 `greedy_stdma` 与 `greedy_stdma_2hop` 的对照动机，写作时强调冲突图、空间复用和集中/启发式调度思想。
- Z-MAC：用于支撑 `zmac_inspired` 的混合 CSMA/TDMA 与 owner-slot 思想。
- TRAMA：用于支撑 `trama_inspired` 的流量感知、邻居信息和时隙分配思想。
- DLMA/DQN-MAC：更适合作为 related work；若论文需要更强 RL 同类实验，再实现轻量 DQN access baseline。
- B-MAC/S-MAC/T-MAC：由于当前项目没有真实能耗模型，适合 related work，不优先进入主实验表。

## 论文写法约束

- 不把 v4 写成全面优于 v3。
- 不把 `zmac_like`、`trama_like`、`zmac_inspired`、`trama_inspired` 写成原论文协议的 bit-level 复现。
- 不做能耗优越性结论，除非后续补齐真实能耗模型。
- 论文主张应聚焦：在 MANET/UAV-inspired 动态拓扑与队列压力下，RL 调度比静态/启发式 TDMA 更能适应负载；饥饿惩罚可改善 fairness，但存在吞吐权衡。
