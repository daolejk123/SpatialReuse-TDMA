# 强化学习论文调研与算法映射

> 目的：把已核验论文映射到当前 `B` / `B_masked_*` 算法结构，区分“值得直接实现”“只借鉴局部模块”“主要用于 related work”三类用途。  
> 校验依据：见 [引用验证-refchecker报告.md](引用验证-refchecker报告.md)。

## 一、当前算法结构摘要

当前主线并不是纯 PPO，而是一个混合调度器：

- [rl/rl_agent.py](../rl/rl_agent.py)：LSTM Actor-Critic、逐时隙动作、two-hop action mask、逐时隙奖励。
- [rl/ppo_trainer.py](../rl/ppo_trainer.py)：PPO 更新、service debt、starvation penalty、动态预算、密度与情境门控。
- [rl/rl_receiver.py](../rl/rl_receiver.py)：局部观测、网络密度、队列/公平性特征解析。
- [DynamicTDMA.cc](../DynamicTDMA.cc)：启发式申请概率、RL 乘数融合、空间复用和逐帧管道输出。

因此，后续调研最有价值的论文应优先回答 5 个问题：

1. 公平性与饥饿是否应从 reward shaping 升级成显式约束。
2. service debt 是否应从手工规则升级成状态变量或对偶变量。
3. two-hop mask 是否有更稳妥的 policy-gradient 处理方式。
4. 独立 PPO 是否应升级为更明确的 multi-agent / CTDE 训练。
5. 当节点数和拓扑继续变化时，是否需要从 one-hot + 手工密度特征转向图表示。

## 二、优先级结论

### 最值得先做的 4 个方向

| 优先级 | 方向 | 直接对应论文 | 对当前项目的意义 |
|---|---|---|---|
| P0 | 显式公平性约束 | ACPO；Triple-Q | 把 `starvation_penalty`、`service_debt_reward` 从 reward engineering 升级为长期平均约束。 |
| P0 | 状态增强的 service debt | Wireless Link Scheduling with State-Augmented GNNs | 让“长期未服务”进入策略状态，而不是只靠固定 boost 触发。 |
| P1 | 多智能体训练 | MAPPO；MADDPG；Multi-Cell Spectrum and Power Allocation | 解决独立 PPO 面对节点间耦合时的非平稳性。 |
| P1 | 图结构与规模泛化 | Large-Scale Graph RL；Graph RL for Radio Resource Allocation | 为更大节点数、动态图和跨规模泛化预留架构路线。 |

### 当前最值得做的下一个版本

如果只做一个新版本，最合理的是：

`B_constrained_debt`

- 保留现有启发式乘数、two-hop mask 和逐时隙奖励。
- 新增显式长期约束：例如最小服务频率、starvation ratio 或 `Wt_P95` 上界。
- 把 service debt / dual variable 作为策略输入，弱化固定 `reward_coef` 和手工 gate。
- 先继续用 PPO 主干；等约束版本跑稳后，再决定是否上 CTDE / MAPPO。

| **Multi-Agent Actor-Critic for Mixed Cooperative-Competitive Environments** | CTDE；集中式 critic；缓解多智能体非平稳性 | **值得借鉴，不必照搬 MADDPG**。你的离散逐时隙 Bernoulli 动作更适合 PPO 主干。 | `rl/ppo_trainer.py` |
| **Multi-Agent Reinforcement Learning for Multi-Cell Spectrum and Power Allocation** | 分布式执行；邻居信息交换；循环网络；5 分位用户性能 | **值得实现为结构参考**。很适合支持“局部执行 + 训练期更多全局信息”的路线。 | `rl/ppo_trainer.py`，`rl/rl_receiver.py` |
| **Large-Scale Graph Reinforcement Learning in Wireless Control Systems** | GNN 策略；图置换不变性；跨规模迁移 | **中长期值得实现**。当 `NodeID one-hot` 成为瓶颈时优先考虑。 | `rl/rl_agent.py`，`rl/rl_receiver.py` |
| **Graph Reinforcement Learning for Radio Resource Allocation** | 图状态；拓扑先验；链路调度 | **值得保留为次级图方法参考**。更偏无线资源分配，不必优先于上一个 GNN 方向。 | `rl/rl_agent.py` |
| **A Deep Reinforcement Learning Framework for Contention-Based Spectrum Sharing** | 分散式 POMDP；长期比例公平；recurrent Q-learning | **值得借鉴 reward / fairness 设计**。和你的 MAC 接入任务相近，但动作粒度不同。 | `rl/rl_agent.py`，`rl/ppo_trainer.py` |
| **Deep Recurrent Q-Learning for Partially Observable MDPs** | LSTM 处理部分可观测 | **已被当前结构吸收，主要用于论文支撑**。 | `rl/rl_agent.py` |
| **Resource Management in Wireless Networks via Multi-Agent Deep Reinforcement Learning** | 邻居观测交换；可扩展多智能体资源调度 | **可作为 related work + 结构参考**。比直接实现更适合支持 CTDE 叙事。 | `rl/rl_receiver.py`，`rl/ppo_trainer.py` |
| **Deep-Reinforcement Learning Multiple Access for Heterogeneous Wireless Networks** | RL-MAC / 接入控制基线；异构接入适应 | **主要用于 related work 或弱 RL baseline 灵感**。与你当前“时隙申请概率乘数”结构不完全一致。 | `docs/论文指标与性能对比基准.md`，可选新增 baseline |
| **Distributed Reinforcement Learning for scalable wireless medium access in IoTs and sensor networks** | 分布式学习；两跳局部信息；公平带宽分配 | **主要用于 related work**。和你的局部观测路线接近，但对下一轮代码改动的直接启发较少。 | `docs/论文指标与性能对比基准.md` |

## 四、按模块看的借鉴关系

### 1. `two-hop mask`

- 首选论文：`A Closer Look at Invalid Action Masking in Policy Gradient Algorithms`
- 当前状态：项目已有 `compute_action_mask()`，且 `B_masked*` 已跑过 pilot。
- 建议动作：
  - 增加 `soft_mask_penalty` 消融，而不是只保留硬屏蔽。
  - 记录被 mask 的概率质量，判断收益来自“学得更好”还是“被先验强行裁剪”。

### 2. `service debt / starvation`

- 首选论文：`ACPO`、`Triple-Q`、`Wireless Link Scheduling with State-Augmented GNNs`
- 当前状态：已有 `service_debt_*`、`starvation_penalty_*` 和若干 gate，但仍然是手工规则。
- 建议动作：
  - 把 `frames_since_tx`、debt level、最小服务缺口作为显式状态。
  - 用 dual variable / virtual queue 统一表示长期未服务压力。
  - 先做约束版 PPO，不要继续无限加 reward 系数。

### 3. `multi-agent`

- 首选论文：`MAPPO`、`MADDPG`、`Multi-Cell Spectrum and Power Allocation`
- 当前状态：每节点独立网络，节点间耦合只通过环境和局部观测体现。
- 建议动作：
  - 先做共享 centralized critic，actor 仍保持本地执行。
  - 评估共享 actor 是否优于当前每节点独立网络。
  - 主指标除总吞吐外，加入 5 分位节点吞吐或 tail fairness。

### 4. `graph / topology`

- 首选论文：`Large-Scale Graph RL`、`Wireless Link Scheduling with State-Augmented GNNs`
- 当前状态：节点身份主要靠 one-hot，拓扑主要靠手工密度和 two-hop 摘要。
- 建议动作：
  - 只有当需要跨 `N` 泛化或更大网络时再上 GNN。
  - v1 可先把邻接关系和 slot 冲突关系导出成图特征，不急着立刻替换 PPO 主干。

### 5. `partial observability`

- 首选论文：`DRQN`、`A Deep RL Framework for Contention-Based Spectrum Sharing`
- 当前状态：LSTM 已经合理；你的问题不是“要不要 LSTM”，而是“LSTM 是否吃到了足够有用的历史状态”。
- 建议动作：
  - 保留 LSTM。
  - 把 debt、mask 历史、请求成功率 EWMA 纳入序列输入，比继续堆更深网络更有价值。

## 五、建议实施顺序

1. **先做 `B_constrained_debt`**
   - 目标：减少手工 reward 系数，显式约束 starvation / 最小服务。
   - 主要改动：`rl/ppo_trainer.py`、`rl/rl_agent.py`。

2. **再做 `B_mappo_constrained`**
   - 目标：验证 centralized critic 是否能缓解节点间非平稳性。
   - 主要改动：`rl/ppo_trainer.py`。

3. **最后再评估 GNN**
   - 触发条件：节点数扩展、跨规模泛化、或 `NodeID one-hot` 影响实验可解释性。
   - 主要改动：`rl/rl_agent.py`、`rl/rl_receiver.py`。

## 六、引用注意事项

- `ACPO`：正式会议版本是 ICML 2024；`refchecker` 识别到的是 2023 年 arXiv 版本，正式写论文时按会议版本引用。
- `Distributed Reinforcement Learning for scalable wireless medium access in IoTs and sensor networks`：`refchecker` 给出的期刊缩写为 `Comput. Networks`，正文写 `Computer Networks` 也可，但参考文献表需统一。
- `Graph Reinforcement Learning for Radio Resource Allocation`：当前校验工具提示 arXiv 最新元数据与早期版本存在标题/作者差异；若正式引用，建议在定稿前再按最终出版版本统一一次。
- 本文档只判断“是否适合你的算法结构”，不把任何论文结果直接当作与你当前 OMNeT++ 实验可数值对比的 benchmark。

## 七、计算机领域高质量补充

这些论文不一定来自无线网络，但方法质量高，且能补当前项目最薄弱的几块。

| 论文 | 为什么值得看 | 可迁移到当前项目的点 | 是否值得实现 | 对应仓库文件 |
|---|---|---|---|---|
| **Constrained Policy Optimization** | ICML 2017，约束 RL 的经典起点之一 | 把 reward 与 cost 分开，显式控制长期约束 | **值得优先读，作为 ACPO 的前置理论** | `rl/ppo_trainer.py` |
| **Counterfactual Multi-Agent Policy Gradients** | AAAI 2018，解决 cooperative MARL 中的 credit assignment | 如果以后使用共享团队奖励，可用 counterfactual baseline 判断“哪个节点导致了收益/冲突变化” | **值得借鉴，优先级低于 centralized critic** | `rl/ppo_trainer.py` |
| **QMIX** | ICML 2018，CTDE / value factorization 代表作 | 若你未来想用全局目标训练、局部动作执行，它给出可分解 joint value 的思路 | **值得作为备选路线，不建议现在改成 value-based 主干** | `rl/ppo_trainer.py` |
| **A Distributional Perspective on Reinforcement Learning** | ICML 2017，distributional RL 代表作 | 从只学均值转向学回报分布，为 tail latency / starvation 风险建模提供理论入口 | **值得借鉴**，可服务风险敏感版本 | `rl/ppo_trainer.py` |
| **Near-Minimax-Optimal Risk-Sensitive Reinforcement Learning with CVaR** | ICML 2023，风险敏感 RL 的高质量理论工作 | 可把 `Wt_P95`、尾部等待、最差节点服务体验从“附属指标”变成优化目标 | **值得借鉴，不建议直接照搬 tabular 算法** | `rl/ppo_trainer.py`，`docs/论文指标与性能对比基准.md` |

### 计算机方向的直接启发

1. `CPO / ACPO` 负责告诉你：公平性和长期服务约束不应永远藏在 reward 里。
2. `COMA / QMIX` 负责告诉你：一旦把全局公平或全局吞吐当目标，多节点 credit assignment 会成为真正问题。
3. `Distributional RL / CVaR RL` 负责告诉你：只看平均吞吐不够，尾部时延和尾部饥饿也可以直接进入目标函数。

## 八、金融领域高质量补充

这些论文不是让你去做金融，而是借它们的目标设计思想来改你的调度器。

| 论文 | 核心思想 | 对当前项目的启发 | 是否值得实现 | 对应仓库文件 |
|---|---|---|---|---|
| **Portfolio Selection** | 均值-方差权衡 | 不要只优化平均 goodput；可以把吞吐均值、吞吐波动、等待波动拆开报告 | **值得作为指标设计思想** | `docs/论文指标与性能对比基准.md` |
| **Optimization of Conditional Value-at-Risk** | 直接优化尾部损失 | 对应 `Wt_P95`、starvation、最差节点等待；适合把“最坏一批节点”纳入目标 | **值得实现局部思想** | `rl/ppo_trainer.py` |
| **Conditional Value-at-Risk for General Loss Distributions** | CVaR 对离散/采样分布仍可稳定处理 | 你的仿真数据天然是离散样本流，CVaR 比只看最大值或 P95 更可优化 | **值得实现局部思想** | `rl/ppo_trainer.py`，指标汇总脚本 |
| **A New Interpretation of Information Rate** | 对数效用 / 长期增长率思想 | 和 proportional fairness 很接近；可支持对 `log throughput` 或按节点 log utility 的奖励设计 | **值得借鉴** | `rl/ppo_trainer.py` |
| **Lifetime Portfolio Selection under Uncertainty: The Continuous-Time Case** | 动态环境中的跨期最优决策 | 支持“当前少抢一些 slot，换未来更稳的队列与公平”这类非近视策略叙事 | **主要用于方法论支撑** | `docs/算法改进记录.md` |

### 金融方向的直接启发

1. **均值之外，还要管波动和尾部。**  
   你现在已经报告 `Wt_P95`、starvation、queue slope，这和金融里“不能只看期望收益”是同一逻辑。

2. **CVaR 比单点 P95 更像可优化目标。**  
   如果后续做风险敏感版本，建议优先考虑“尾部平均损失”而不是只继续堆单点阈值。

3. **对数效用可以给 fairness 一个更自然的目标。**  
   这比继续手工调 `starvation_penalty_coef` 更有理论感，尤其适合“总吞吐与节点间公平都要兼顾”的论文叙事。

## 九、跨领域后的推荐路线

如果把无线、计算机、金融三组文献合起来看，当前最有前景的路线已经比较清楚：

1. **`B_constrained_debt`**  
   - 来自无线文献的 service debt。  
   - 来自计算机文献的 constrained RL。  
   - 来自金融文献的 CVaR / tail-risk 目标。

2. **`B_constrained_debt_cvar`**  
   - 在上一步基础上，引入尾部等待或尾部饥饿目标。  
   - 这会比继续调固定 `wt_threshold` 更像一个真正的新方法。

3. **`B_mappo_constrained`**  
   - 若前两个版本有效，再把 centralized critic、credit assignment 引入。  
   - 这时 `COMA / QMIX / MAPPO` 的价值才会真正体现。
